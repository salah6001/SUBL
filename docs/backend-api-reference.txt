=======================================================================
  Subl - Back-end API Reference
=======================================================================

REST/JSON API exposed by the .NET 10 Web.Api project. Every row is one
endpoint: HTTP METHOD, ROUTE, and what it does.

Conventions
  - Routes are relative to the API host root.
  - {id:guid} etc. are route parameters.
  - Auth column:
        (auth)  = requires a valid JWT (default for almost everything)
        (admin) = requires super-admin / elevated permission
        (public)= no authentication required
  - The contract is also published as OpenAPI and browsable via Scalar.

Endpoint count: ~140, grouped by feature area below.

-----------------------------------------------------------------------
RESPONSE MODEL (all endpoints)
-----------------------------------------------------------------------
Success:  HTTP 200/201 with a JSON body (or 204 No Content).
Failure:  HTTP 4xx/5xx as RFC-7807 problem details, produced by mapping
          the handler's Result.Failure(error) to the right status code.
          Validation failures return 400 with the offending fields.
Auth:     JWT bearer token in the Authorization header; refresh via
          POST users/refresh.
